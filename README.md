# Домашнє завдання 6: Робота з графами

## Завдання 1: Створення та аналіз графа

Створено граф міської транспортної мережі з 10 станцій метро.

**Основні характеристики:**
- Кількість станцій (вершин): 10
- Кількість з'єднань (ребер): 15
- Середній ступінь вершини: 3.0
- Центральна станція (Central Station) має найбільший ступінь: 4 з'єднання

## Завдання 2: Порівняння алгоритмів DFS та BFS

### Результати пошуку шляху від "Central Station" до "Stadium"

**DFS (Пошук у глибину):**
```
Central Station -> North Terminal -> Airport -> Stadium
Довжина: 4 станції
```

**BFS (Пошук у ширину):**
```
Central Station -> East Plaza -> Stadium
Довжина: 3 станції
```

### Порівняння та аналіз

| Характеристика | DFS | BFS |
|----------------|-----|-----|
| Довжина шляху | 4 станції | 3 станції |
| Знайдений шлях | Довший | **Найкоротший** |
| Принцип роботи | Йде вглиб першою гілкою | Перевіряє всі сусідні вершини |

### Пояснення відмінностей

**Чому DFS знайшов довший шлях?**

DFS використовує принцип "йди якомога глибше". Алгоритм:
1. Почав з Central Station
2. Вибрав першого сусіда (North Terminal)
3. Продовжив углиб до Airport
4. Потім до Stadium

DFS не гарантує найкоротший шлях, він просто знаходить **будь-який** шлях до цілі.

**Чому BFS знайшов коротший шлях?**

BFS використовує принцип "перевір усіх сусідів на поточному рівні". Алгоритм:
1. Почав з Central Station
2. Перевірив **всіх** сусідів першого рівня (North Terminal, South Terminal, East Plaza, West Park)
3. Знайшов, що East Plaza має пряме з'єднання зі Stadium
4. Повернув найкоротший шлях

BFS **завжди** знаходить найкоротший шлях у незваженому графі.

### Висновки

1. **BFS оптимальніший** для пошуку найкоротшого шляху в незваженому графі
2. **DFS швидший** для пошуку будь-якого шляху (не обов'язково найкоротшого)
3. **BFS використовує більше пам'яті** (зберігає всю чергу вершин)
4. **DFS ефективніший за пам'яттю** (використовує стек/рекурсію)

### Практичне застосування

- **BFS**: навігація (Google Maps), соціальні мережі (найближчі друзі)
- **DFS**: пошук виходу з лабіринту, аналіз файлової системи
